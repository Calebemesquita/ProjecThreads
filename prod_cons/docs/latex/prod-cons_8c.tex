\doxysubsection{Descrição detalhada}
Simulação do problema Produtor-\/\+Consumidor usando pthreads, semáforos e variáveis de condição. 

Este programa implementa uma solução para o problema clássico do produtor-\/consumidor. Ele simula um cenário com múltiplos "{}caixas"{} (produtores) que geram vendas (valores de ponto flutuante) e as colocam em um buffer circular compartilhado. Um único "{}gerente"{} (consumidor) aguarda até que o buffer esteja completamente cheio para então processar todas as vendas de uma vez, calculando o valor médio.

A sincronização entre as threads é gerenciada da seguinte forma\+:
\begin{DoxyItemize}
\item {\bfseries{Mutex ({\ttfamily mutex})\+:}} Garante o acesso exclusivo às seções críticas, protegendo o buffer e as variáveis compartilhadas ({\ttfamily count}, {\ttfamily in\+\_\+idx}, {\ttfamily out\+\_\+idx}, {\ttfamily active\+\_\+producers}) contra condições de corrida.
\item {\bfseries{Semáforos ({\ttfamily empty\+\_\+slots}, {\ttfamily full\+\_\+slots})\+:}} {\ttfamily empty\+\_\+slots} controla o número de posições vazias no buffer, fazendo com que os produtores esperem se o buffer estiver cheio. {\ttfamily full\+\_\+slots} foi mantido para ilustrar a solução clássica, embora o consumidor neste exemplo específico não espere por um único item.
\item {\bfseries{Variável de Condição ({\ttfamily buffer\+\_\+full\+\_\+cond})\+:}} Permite que o consumidor (gerente) espere de forma eficiente sem consumir CPU ({\ttfamily pthread\+\_\+cond\+\_\+wait}) até que o buffer esteja cheio ou que todos os produtores tenham terminado seu trabalho. Os produtores sinalizam ({\ttfamily pthread\+\_\+cond\+\_\+signal}) quando o buffer enche, e um {\ttfamily broadcast} é usado no final para garantir que o consumidor acorde e termine. 
\end{DoxyItemize}

{\footnotesize\ttfamily \label{prod-cons_8c_a6c0ce21bf6fb29cd5d2d0b367f315da1} 
void \texorpdfstring{$\ast$}{*} consumer (\begin{DoxyParamCaption}\item[{void \texorpdfstring{$\ast$}{*}}]{args}{}\end{DoxyParamCaption})}



Função executada pela thread consumidora (gerente). 

O consumidor entra em um loop infinito para processar as vendas. Ele bloqueia o mutex e aguarda na variável de condição ({\ttfamily pthread\+\_\+cond\+\_\+wait}) até que o buffer esteja cheio ({\ttfamily count == BUFFER\+\_\+\+SIZE}) ou não haja mais produtores ativos ({\ttfamily active\+\_\+producers == 0}). Quando acordado e a condição é satisfeita, ele processa {\itshape todos} os itens presentes no buffer, calculando a soma e a média. Em seguida, ele zera o contador de itens e libera os slots correspondentes no semáforo {\ttfamily empty\+\_\+slots}. O loop termina quando não há mais produtores ativos e o buffer está vazio.

\doxysubsubsection{\texorpdfstring{producer()}{producer()}}
{\footnotesize\ttfamily \label{prod-cons_8c_a13f485a0cfd6e52c071bef0b7adb70cf} 
void \texorpdfstring{$\ast$}{*} producer (\begin{DoxyParamCaption}\item[{void \texorpdfstring{$\ast$}{*}}]{args}{}\end{DoxyParamCaption})}



Função executada pelas threads produtoras (caixas). 

Cada produtor gera um número pré-\/definido de vendas com valores aleatórios. Para cada venda, ele aguarda por um slot vazio no buffer ({\ttfamily sem\+\_\+wait}), bloqueia o mutex, adiciona o valor da venda ao buffer, atualiza os contadores e o índice de entrada. Se o buffer ficar cheio após a inserção, ele sinaliza a variável de condição {\ttfamily buffer\+\_\+full\+\_\+cond} para acordar o gerente. Após produzir todas as suas vendas, decrementa o contador {\ttfamily active\+\_\+producers} e, se for o último produtor a terminar, envia um {\ttfamily broadcast} na variável de condição para garantir que o consumidor processe os itens restantes e termine.
